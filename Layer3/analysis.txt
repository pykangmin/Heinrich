## 증상
scoring.engine 첫 실행 후 daily_scores 상위 20개 종목의 score_bottleneck이
전부 100으로 동일. total_heinrich 변별력 사실상 Track A(score_original)에만 의존.

## 원인 분석

### 데이터 현황
- tickers 503개 중 vc_stage_num 배정: 395개 (108개는 NULL → 50 처리, 정상)
- (vc_stage_num, vc_position) 그룹 멤버 수 분포 조회 결과:
  상위 20개 그룹이 전부 cnt=1 (단일 종목 그룹)

### 인과관계
1. _pct_high_better: rank(pct=True, method='average')
   → 그룹 멤버가 1개이면 항상 rank=1.0 → 백분위 점수 = 100
2. 단일 멤버 그룹이 다수 → score_bottleneck=100인 종목이 대량 발생
3. _minmax_0_100: 최솟값과 최댓값이 모두 100인 종목이 많아
   MinMax 후에도 상위권 전체가 100으로 유지됨

### 결론
(vc_stage_num, vc_position) 복합 그룹은 분류.json의 포지션 라벨이
종목별로 지나치게 세분화되어 있어 통계적으로 유의미한 비교 집단을 구성하지 못함.

## 수정 방향

MIN_GROUP_SIZE = 3 기준 폴백 체계 도입:

  1순위: (vc_stage_num, vc_position) 그룹 멤버 ≥ 3 → 복합 그룹 백분위
  2순위: vc_stage_num 단독 그룹 멤버 ≥ 3 → stage 단독 백분위
  3순위: 전체 유니버스 백분위 (그룹 어디도 ≥ 3 불충족 시)
  예외:  vc_stage_num, vc_position 둘 다 NULL → score_bottleneck = 50

구현: df에 그룹별 멤버 수를 groupby.transform('count')로 미리 계산 후
_group_key 임시 컬럼을 조건에 따라 할당 → _pct_high_better 단일 호출.
